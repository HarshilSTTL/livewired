### [2026-04-08 21:30] | FIX | Timezone conversion — use event_timezone instead of hardcoded UTC + profile_id in get_event_list

**Root cause:**
Read SPs were doing `AT TIME ZONE 'UTC'` but `event_date`/`event_time` store the creator's
local time. This caused a wrong offset shift (e.g. IST time treated as UTC → off by 5:30h).

**Fix:**
Replaced hardcoded `'UTC'` with `e.event_timezone` in all conversion expressions:
```sql
-- Before: (stored::timestamp AT TIME ZONE 'UTC') AT TIME ZONE p_timezone
-- After:  (stored::timestamp AT TIME ZONE e.event_timezone) AT TIME ZONE p_timezone
```

**create_event / update_event:**
Removed UTC conversion — now stores `p_event_date` and `p_event_time` as-is (creator's local time)
alongside `event_timezone`. Simpler and consistent with how data was already being stored.

**process_event_reminders:**
Updated cron comparison to use `AT TIME ZONE e.event_timezone` so reminder fires at correct UTC moment.

**Also added:** `profile_id` field to all 3 branches of `get_event_list` response.

**Files changed:**

### [2026-04-08 14:00] | SCHEMA + SP | Timezone support for events — UTC storage + per-viewer conversion

**Why:**
Creators schedule events in their local timezone (e.g. USA). Without timezone support, a 8pm USA event shows as 8pm for everyone — wrong for all other timezones. Events must shift date AND time based on each viewer's timezone.

**Approach:**
- Store all event datetimes in UTC in `event_mst`
- Creator sends their IANA timezone on create/update → SP converts to UTC before storing
- Viewer sends their IANA timezone on every read → SP converts UTC → viewer's local date + time

**Schema change (run in Supabase):**
```sql
ALTER TABLE public.event_mst
ADD COLUMN IF NOT EXISTS event_timezone text NOT NULL DEFAULT 'UTC';
```
Existing rows default to `'UTC'` — safe as stored values had no offset context.

**SP changes:**
- `create_event` — `+ p_timezone`, converts creator local → UTC on parent insert + every child occurrence (handles DST per occurrence)
- `update_event` — `+ p_timezone`, recalculates UTC if date/time is being changed
- `get_event_list` — `+ p_timezone`, all date/time comparisons + output in viewer's local
- `get_event_by_id` — `+ p_timezone`, output in viewer's local
- `get_profile_events` — `+ p_timezone`, week window filter + output in viewer's local
- `get_event_dates` — `+ p_timezone`, month filter + grouping in viewer's local date
