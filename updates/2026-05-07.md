**1. FIX — Collaborator permissions corrected in `update_event` and `delete_event`**

- **Before:** Both SPs accepted accepted collaborators as a second valid ownership path, granting them update and delete rights.
- **After:** Both SPs are **owner-only**. The collaborator branch (`NOT EXISTS (... event_collaborators ...)`) has been removed entirely. A collaborator calling either SP receives `"Event not found or access denied"`.
- **Why:** The requirement explicitly states collaborators must not have update, delete, or postpone permissions.

---

**2. SP — `create_event` now accepts `p_collaborator_ids uuid[] DEFAULT null`**

New parameter added after `p_is_collaborative`:

| New param | Type | Default | Notes |
|---|---|---|---|
| `p_collaborator_ids` | `uuid[]` | null | Profile IDs to invite at creation time. Requires `p_is_collaborative = true`. Max 5. |

**Validation (before event insert):**
- If `p_collaborator_ids` is non-empty AND `p_is_collaborative = false` → error: `"Cannot add collaborators when is_collaborative is false"`

**Invite loop (after event insert + recurring generation):**

For each ID in `p_collaborator_ids`:
- Skip if ID = `p_profile_id` (cannot invite self)
- Skip if `v_collab_count >= 5` (hard cap)
- Skip if profile not found or not active
- Otherwise: INSERT into `event_collaborators` (status = `'pending'`) + INSERT notification for invitee

Skipped IDs are collected and returned in `data.skipped_collaborator_ids` (always present, empty array `[]` when nothing was skipped).

**Updated RETURN:**
```json
{
  "status":  true,
  "message": "Event created successfully",
  "data": {
    "event_id":                 "uuid",
    "skipped_collaborator_ids": []
  }
}
```

**3. SP — `search_collaborator_profiles` (new, SP #25)**

- **Endpoint:** `POST /rpc/search_collaborator_profiles`
- **Purpose:** Profile search scoped to the collaborator picker. Always excludes the creating/inviting profile. Optionally excludes already-invited profiles for a given event.

**Parameters:**

| Param | Type | Required | Notes |
|---|---|---|---|
| `p_keyword` | text | ✅ | Min 2 chars. Matched against `profile_name` and `bio` |
| `p_exclude_profile_id` | uuid | ✅ | The creating profile — always excluded from results |
| `p_event_id` | uuid | ❌ | If provided, also excludes profiles with pending/accepted active invites for this event |
| `p_limit` | int | ❌ | Default 20 |

**Exclusion logic:**
- `id != p_exclude_profile_id` — always applied
- `id NOT IN (SELECT profile_id FROM event_collaborators WHERE event_id = p_event_id AND status IN ('pending','accepted') AND is_deleted = false)` — applied only when `p_event_id` is provided
