### [2026-06-01 14:45] | FIX | Add LATERAL keyword to fix correlated subquery scope errors #sql-fix #critical

**Critical Fix:** Added LATERAL keyword to all derived table subqueries to explicitly allow correlation with outer `creator_profiles cp` table.

**Root Cause:**
- Functions that iterate over multiple profiles (search_profiles, search_collaborator_profiles, get_profile_by_userid, get_profiles) use correlated subqueries
- PostgreSQL parser fails when derived tables reference outer table aliases without LATERAL keyword
- This caused "missing FROM-clause entry for table 'cpa'" error

**Fix Applied:**
Changed all instances from:
```sql
FROM (
    SELECT ... WHERE cpa.profile_id = cp.id ...
) alias_name
```

To:
```sql
FROM LATERAL (
    SELECT ... WHERE cpa.profile_id = cp.id ...
) alias_name
```

**Functions Updated:**
1. **search_profiles.md (v2.1)** - Added LATERAL to platforms, additional_links, custom_links subqueries
2. **search_collaborator_profiles.md (v2.1)** - Added LATERAL to platforms, additional_links, custom_links subqueries
3. **get_profile_by_userid.md (v2.1)** - Added LATERAL to platforms, additional_links, custom_links subqueries
4. **get_profiles.md (v2.1)** - Added LATERAL to platforms, additional_links, custom_links subqueries

**Why LATERAL Works:**
- LATERAL explicitly tells PostgreSQL that the derived table can reference columns from preceding FROM items
- Allows the subquery to access `cp.id` from the outer SELECT loop
- Properly scopes all table aliases (cpa, pcl, p, plp) relative to their usage

**Result:**
✅ "missing FROM-clause" SQL errors resolved  
✅ Correlated subqueries now properly execute  
✅ All link groups (platforms, additional_links, custom_links) return correctly  
✅ User preference ordering via array_position() works
