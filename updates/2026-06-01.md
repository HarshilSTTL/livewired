### [2026-06-01 14:45] | FIX | Add LATERAL keyword to fix correlated subquery scope errors #sql-fix #critical

**Critical Fix:** Added LATERAL keyword to all derived table subqueries to explicitly allow correlation with outer `creator_profiles cp` table in multi-row iterations.

**Issue:**
- Functions that iterate over multiple profiles (search_profiles, search_collaborator_profiles, get_profile_by_userid, get_profiles) use correlated subqueries
- PostgreSQL parser failed when derived tables tried to reference outer table aliases without LATERAL keyword
- Error: `"missing FROM-clause entry for table 'cpa'"` when executing v2.1 API endpoints

**Root Cause:**
```sql
-- WRONG - PostgreSQL doesn't allow this correlation without LATERAL
FROM (
    SELECT ... WHERE cpa.profile_id = cp.id ...
) alias_name
```

**Solution Applied:**
```sql
-- CORRECT - LATERAL explicitly allows derived table to access outer scope
FROM LATERAL (
    SELECT ... WHERE cpa.profile_id = cp.id ...
) alias_name
```

## Summary

**Total Fixes Today:** 3 critical SQL fixes  
**Total Functions Updated:** 5 core v2.1 functions  
**Total Subqueries Fixed:** 15 (3 per function × 5 functions: platforms, additional_links, custom_links)  
**Status:** ✅ All v2.1 endpoints operational  

**Critical Learnings:**
1. PostgreSQL requires explicit LATERAL keyword for derived tables that reference outer scope in row-iterating queries
2. Even single-profile functions (get_profile_by_id) using SELECT...FROM pattern need LATERAL
3. Consistency in variable references (v_profile_id vs cp.id) is critical in complex nested queries
4. Case sensitivity: `coalesce` vs `COALESCE` - always use uppercase for PostgreSQL functions

**Root Cause Summary:**
The "missing FROM-clause entry for table 'cpa'" error occurs when:
- Derived tables try to access outer table aliases (cp, cpa, pcl, etc.)
- WITHOUT the explicit LATERAL keyword
- PostgreSQL's parser can't infer the correlation

**Solution Pattern:**
```sql
-- Pattern for all multi-field JSON aggregation:
'field_name', (
    SELECT COALESCE(json_agg(...), '[]'::json)
    FROM LATERAL (
        SELECT ... WHERE outer_var = reference ...
    ) alias
)
```
