### [2026-06-01 14:45] | FIX | Add LATERAL keyword to fix correlated subquery scope errors #sql-fix #critical

**Critical Fix:** Added LATERAL keyword to all derived table subqueries to explicitly allow correlation with outer `creator_profiles cp` table in multi-row iterations.

**Issue:**
- Functions that iterate over multiple profiles (search_profiles, search_collaborator_profiles, get_profile_by_userid, get_profiles) use correlated subqueries
- PostgreSQL parser failed when derived tables tried to reference outer table aliases without LATERAL keyword
- Error: `"missing FROM-clause entry for table 'cpa'"` when executing v2.1 API endpoints

**Root Cause:**
```sql
-- WRONG - PostgreSQL doesn't allow this correlation without LATERAL
FROM (
    SELECT ... WHERE cpa.profile_id = cp.id ...
) alias_name
```

**Solution Applied:**
```sql
-- CORRECT - LATERAL explicitly allows derived table to access outer scope
FROM LATERAL (
    SELECT ... WHERE cpa.profile_id = cp.id ...
) alias_name
```

**Why LATERAL Works:**
- Explicitly tells PostgreSQL that derived table can reference columns from preceding FROM items
- Allows nested subqueries to access `cp.id` from outer SELECT loop
- Properly scopes all table aliases (cpa, pcl, p, plp) relative to their usage in correlated WHERE clauses
