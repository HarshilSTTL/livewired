### [2026-04-06 11:00] | FIX | username — removed UNIQUE constraint from users table

**What changed:**
`username` on `users` table changed from UNIQUE to non-unique. Multiple users can have the same username. Uniqueness check removed from all SPs that previously enforced it.

**Migration SQL (run in Supabase SQL Editor):**
```sql
DROP INDEX IF EXISTS users_username_key;
```

**SP changes:**
- `signup`, `register`, `google_auth` — removed "Username already taken" uniqueness check

**Files changed:**
- [[schema/tables/02_users.md]] — removed UNIQUE, updated migration note

### [2026-04-06 10:00] | FIX | signup — params renamed to p_ prefix

**What changed:**
`email`, `password`, `ip` params clashed with `users` table column names causing
`"column reference is ambiguous"` error on every signup call.

Renamed: `email → p_email`, `password → p_password`, `ip → p_ip`

**Flutter call update required:**
```dart
await supabase.rpc('signup', params: {
  'p_email':    email,
  'p_password': password,
  'p_username': username,
});
```

**Files changed:**
- `functions/auth/signup.md` — param names updated
- `docs/api/auth/signup.md` — param table + request example updated
