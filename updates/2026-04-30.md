### [2026-04-30 00:10] | SCHEMA | Add optional event_end_time to events + revise Live list logic

**Reason:**
Events can optionally have an end time. The dashboard Live section should only show events that have an end time and are currently ongoing. Started events with no end time should stay in Today (not Live).

**Schema Change:**
Added `event_end_time time NULL` to `event_mst`.

**SPs Updated:**
- `create_event` — added param `p_event_end_time` (optional) + validation (`end_time > start_time`) + persisted to parent + recurring children
- `update_event` — added param `p_event_end_time` (optional) + validation on final start/end time + persisted to parent + recurring children
- `get_event_list` — Live list now requires `event_end_time` and uses `NOW()` between start/end; Today includes upcoming + started-without-end_time
- `get_profile_events` / `get_event_by_id` — return `event_end_time`

### [2026-04-30 00:00] | SP | Add default platform toggles to get_user_profiles

**Reason:**
The post-login profile picker needs to know each profile’s default platform preferences so the app can initialize Twitch/Kick toggles immediately after selection (without requiring an extra full-profile fetch).

**Change:**
`get_user_profiles` now returns two additional boolean fields per profile:
- `p_twitch_by_default`
- `p_kick_by_default`
