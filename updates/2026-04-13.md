### [2026-04-13 01:00] | SP | Platform & Custom Link CRUD — 7 new stored procedures

**What changed:**
Built 7 separate CRUD SPs for managing platform links (Section 1 + 2) and custom links (Section 3) on the profile edit screen. Each operation has its own dedicated SP — no bulk replace, no delete-all-and-recreate.

**SPs added:**

| SP | Endpoint | Operation | Table |
|---|---|---|---|
| `get_profile_platforms` | POST /rpc/get_profile_platforms | Read all active platform links | `creator_platform_accounts` |
| `add_profile_platform` | POST /rpc/add_profile_platform | Insert one platform link | `creator_platform_accounts` |
| `update_profile_platform` | POST /rpc/update_profile_platform | Update channel_url by row id | `creator_platform_accounts` |
| `delete_profile_platform` | POST /rpc/delete_profile_platform | Soft-delete by row id | `creator_platform_accounts` |
| `add_custom_link` | POST /rpc/add_custom_link | Insert one custom link | `profile_custom_links` |
| `update_custom_link` | POST /rpc/update_custom_link | Update name + URL by row id | `profile_custom_links` |
| `delete_custom_link` | POST /rpc/delete_custom_link | Soft-delete by row id | `profile_custom_links` |

**Design notes:**
- All write SPs verify ownership via `creator_profiles.user_id`
- All deletes are soft (`is_deleted = true`, `deleted_at = now()`)
- `get_profile_platforms` returns the `creator_platform_accounts.id` needed as `p_id` for update/delete calls
- `get_profile_custom_links` (existing SP) returns the `profile_custom_links.id` needed for custom link update/delete calls

Added `is_deleted` and `deleted_at` columns to `creator_platform_accounts` to support soft delete.
