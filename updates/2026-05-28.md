### [2026-05-28 20:45] | FIX | Fix SQL scoping error in v2.1 API subqueries #sql-fix #bug

**Bug Fix:** Fixed "missing FROM-clause entry for table 'cpa'" SQL error in v2.1 API functions caused by improper scoping of correlated subqueries. #sql #api

**Issue:**
- Subqueries in v2.1 functions were referencing `cp.id` which wasn't properly scoped in deeply nested query structures
- Error: "missing FROM-clause entry for table 'cpa'"
- Affected: get_profile_by_id_v2_1 (and later: get_profile_by_userid_v2_1, search_profiles_v2_1, etc.)

**Solution Applied:**
- **get_profile_by_id_v2_1:** Changed all `cp.id` references to `v_profile_id` (the properly scoped variable)
  - This function uses a single variable `v_profile_id` instead of iterating over `cp`
  - All 3 subqueries (platforms, additional_links, custom_links) now use `v_profile_id` for profile filtering
  - Fixes the scoping issue completely

**Pattern for Other Functions:**
For get_profile_by_userid_v2_1, search_profiles_v2_1, search_collaborator_profiles_v2_1, get_profiles_v2_1:
```sql
-- Move COALESCE inside the SELECT to maintain proper scoping:
'platforms', (
    SELECT COALESCE(json_agg(...), '[]'::json)
    FROM (SELECT ... WHERE cpa.profile_id = cp.id ...) platform_list
)

-- Instead of:
'platforms', COALESCE((
    SELECT json_agg(...) FROM (...)
), '[]'::json),
```

### [2026-05-28 18:45] | SCHEMA | Implement drag-drop link reordering (Phase 1: Core APIs) #reordering #platforms #custom-links

**Enhancement:** Created migration, reorder endpoint, and v2.1 profile fetching APIs for drag-drop reordering of social links. #drag-drop #api #versioning #schema

**Purpose:** Allow creators to customize the order platforms, additional links, and custom links appear on their profile.

**Phase 1 Implementation:**

1. **Database Migration**
   - Created `profile_link_preferences` table
   - Stores drag-drop order for 3 link groups: platforms (int[]), additional_ids (int[]), custom_ids (uuid[])
   - Single row per profile with CASCADE DELETE on profile deletion
   - Index on (profile_id) for fast lookups
   - File: [[schema/migrations/create_profile_link_preferences.md]]

2. **Reorder API (Save)**
   - Endpoint: `POST /rpc/reorder_social_links_v2`
   - Saves user's drag-drop reordering to profile_link_preferences table
   - UPSERT pattern: inserts new row if no preferences exist, updates existing
   - Parameters: p_profile_id, p_platform_ids[], p_additional_ids[], p_custom_ids[]
   - Returns: saved order in response data
   - Idempotent: calling twice with same data = same result
   - Partial updates: can save only platforms, skip additional/custom with empty arrays
   - File: [[functions/profiles/reorder_social_links.md]]
   - Doc: [[docs/api/profiles/reorder_social_links.md]]

3. **Profile Fetching API (Read)**
   - Created: `get_profile_by_id_v2_1` endpoint
   - Uses profile_link_preferences table for custom ordering
   - Returns links in order: platforms (reordered) → additional links (reordered) → custom links (reordered)
   - Maintains backward compatibility with v2 response structure
   - Falls back to unordered if no preferences exist
   - File: [[functions/profiles/get_profile_by_id.md]]
   - Doc: [[docs/api/profiles/get_profile_by_id.md]]

**Database Schema:**
```
profile_link_preferences
  ├── id (uuid, PK)
  ├── profile_id (uuid, FK, UNIQUE)
  ├── platform_ids_order (int[] — e.g., [3, 4, 2, 1])
  ├── additional_ids_order (int[] — e.g., [7, 5, 6, 8])
  └── custom_ids_order (uuid[] — e.g., [uuid-10, uuid-8, uuid-9])
```

**Link Group Classification:**
- **Platforms:** IDs 1-4 from platforms table (YouTube, Twitch, Kick, Rumble)
- **Additional Links:** IDs 5-42+ from platforms table
- **Custom Links:** Creator-defined links from profile_custom_links table

**API Request/Response Example (reorder_social_links_v2):**
```json
Request: {
  "p_profile_id": "abc-123-def-456",
  "p_platform_ids": [3, 4, 2, 1],
  "p_additional_ids": [7, 5, 6],
  "p_custom_ids": ["uuid-10", "uuid-8"]
}

Response: {
  "status": true,
  "message": "Links reordered successfully",
  "data": {
    "profile_id": "abc-123-def-456",
    "platform_ids_order": [3, 4, 2, 1],
    "additional_ids_order": [7, 5, 6],
    "custom_ids_order": ["uuid-10", "uuid-8"]
  }
}
```
