### [2026-05-15 12:30] | SP | `clear_notifications` — Clear/hide notifications for authenticated user

**Feature:** New stored procedure to clear (hide) notifications from the user's notification list without deleting them from the database.

**Endpoint:** `POST /rpc/clear_notifications`

**Parameters:**
- `p_user_id` (uuid, required) — Authenticated user's ID
- `p_notification_ids` (uuid[], optional) — Specific notification IDs to clear. NULL = clear all

**Behavior:**
- If `p_notification_ids` IS NULL: Clears ALL notifications for user where `is_cleared = false`
- If `p_notification_ids` provided: Clears only those specific IDs (must belong to user)
- Already-cleared notifications are silently skipped (idempotent)
- Sets `is_cleared = true` on matching rows

**Response:**
```json
{
  "status": true,
  "message": "X notification(s) cleared"
}
```

**Security:** User isolation enforced via `WHERE user_id = p_user_id` filter. Users cannot clear other users' notifications.
