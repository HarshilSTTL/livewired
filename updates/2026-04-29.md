### [2026-04-29 00:00] | SCHEMA | Add twitch_by_default and kick_by_default to creator_profiles

**Reason:**
The Profile Preferences screen has two toggles — "Twitch on by default?" and "Kick on by default?". These are per-profile settings, so they belong directly on the `creator_profiles` table.

**Schema Change:**
Added two new boolean columns to `creator_profiles`:

```sql
ALTER TABLE public.creator_profiles ADD COLUMN IF NOT EXISTS twitch_by_default boolean DEFAULT false;
ALTER TABLE public.creator_profiles ADD COLUMN IF NOT EXISTS kick_by_default   boolean DEFAULT false;
```

Both default to `false` — off unless the creator explicitly enables them.

**SPs Updated:**

| Function | Change |
|---|---|
| `create_profile` | Added `p_twitch_by_default` and `p_kick_by_default` params → inserted into table → returned in response |
| `update_profile` | Added `p_twitch_by_default` and `p_kick_by_default` params → COALESCE update (only updates if passed) |
| `get_profile_by_id` | Returns `twitch_by_default` and `kick_by_default` in response |
| `get_profile_by_username` | Returns `twitch_by_default` and `kick_by_default` in response |
| `get_profile_by_userid` | Returns `twitch_by_default` and `kick_by_default` in response |

**Not changed:**
- `get_user_profiles` — lightweight profile selector, only returns `profile_id`, `profile_name`, `avatar`, `is_default`

**Frontend Impact:**
- On **Save Profile** → send `twitch_by_default` and `kick_by_default` in `update_profile` call
- On **Create Profile** → send both fields in `create_profile` call
- On **Load Profile** → read both fields from any `get_profile` response to pre-fill the toggle states

### [2026-04-29 00:01] | SCHEMA | Remove username column from creator_profiles

**Reason:**
`username` was a globally unique column on `creator_profiles`. Removing it simplifies profile identity — profiles are now identified by `profile_name` and `profile_id` only.

**Schema Change:**

```sql
ALTER TABLE public.creator_profiles DROP COLUMN IF EXISTS username;
```

**Functions Updated:**

| Function | Change |
|---|---|
| `create_profile` | Removed `p_username` param, uniqueness check, username from INSERT and platform account insert |
| `update_profile` | Removed `p_username` param, username validation, `v_final_username` variable, username from UPDATE and platform account upsert |
| `get_profile_by_id` | Removed `username` from response |
| `get_profile_by_userid` | Removed `username` from response |
| `get_profiles` | Removed `username` from search filter, WHERE clause, and ORDER BY |
| `search_profiles` | Removed `username` from response, WHERE clause, and `word_similarity` scoring |
| `get_creators` | Removed `username` from response |

**Deprecated:**
- `get_profile_by_username` — marked as DEPRECATED. Use `get_profile_by_id` instead.

**Frontend Impact:**
- Remove username input field from Create Profile and Edit Profile screens
- Remove `@username` display from all profile cards and views
- Switch any profile navigation that used username to use `profile_id` instead
