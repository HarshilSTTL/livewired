### [2026-05-14] | FIX | `get_event_list` showing duplicate recurring event on start date

**Bug:** When a recurring event was created, `get_event_list` returned 2 entries for today if today was the start date of the recurrence. From the next day onward it correctly returned 1 entry per day.

**Root cause:** `create_event` inserts a parent template row (`parent_event_id = NULL`, `is_recurring = true`) with `event_date = p_event_date`, plus child occurrence rows (one per day). For the start date, both the parent and the Thursday child row had `event_date = today`, so both matched the date filter.

`get_profile_events` already had the correct guard to exclude parent templates:
```sql
AND (e.is_recurring = false OR e.parent_event_id IS NOT NULL)
```
But `get_event_list` was missing this filter in all four of its query blocks (past date, today-live, today-today, future date), so the parent template leaked through.

---

#### [[functions/events/get_event_list.md]]

**FIX — added recurring parent exclusion filter to all four query blocks**

- **Before:** All four `FROM event_mst` queries filtered only on `e.is_deleted = false` and date/status conditions. Recurring parent template rows (those with `is_recurring = true` AND `parent_event_id IS NULL`) were not excluded.
- **After:** Added `AND (e.is_recurring = false OR e.parent_event_id IS NOT NULL)` to each of the four query blocks:
  - PAST DATE → `v_today` query
  - TODAY → `v_live` query
  - TODAY → `v_today` query
  - FUTURE DATE → `v_today` query

This matches the filter already present in `get_profile_events` and correctly suppresses the template row from all list responses.
