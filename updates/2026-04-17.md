### [2026-04-17 00:00] | FIX | get_profiles + get_suggested_profiles — followers count now respects show_followers flag

**Problem:**
Both SPs always returned the live follower count regardless of `show_followers`:
```sql
(SELECT COUNT(*) FROM follows f WHERE f.profile_id = cp.id AND f.is_active = true) AS followers
```
This leaked follower counts for profiles where the creator had disabled follower visibility.

**Fix:**
Wrapped the subquery in a `CASE` expression in both SPs:
```sql
CASE
    WHEN cp.show_followers = true THEN (
        SELECT COUNT(*) FROM follows f
        WHERE f.profile_id = cp.id AND f.is_active = true
    )
    ELSE NULL
END AS followers
```

**Already correct (no change needed):**
- `search_profiles` — had `ELSE null` already
- `get_creators` — had `ELSE null` already
- `get_profile_by_username` — had `ELSE null` already
