### [2026-06-23] | FEAT | Default event end time to 2 hours when not specified #enhancement

**Business rule added:**
If a creator does not provide `p_event_end_time` when creating an event, the system now automatically defaults it to **2 hours after** `p_event_time`.

**Why:**
Events without an end time had no duration context — they couldn't show a "LIVE" status and produced less useful conflict-check results. Defaulting to 2 hours gives every event a sensible duration without forcing the creator to always specify one.

**Behavior:**

| Scenario | Result |
|----------|--------|
| `p_event_end_time` not provided | Defaults to `p_event_time + 2 hours` |
| `p_event_end_time` explicitly provided | Used as-is (unchanged) |
| Start time is `23:00` (wraps past midnight) | End time becomes `01:00` — handled as cross-midnight by existing logic |

**Cross-midnight note:**
PostgreSQL `TIME + INTERVAL '2 hours'` wraps correctly past midnight (e.g. `23:00 → 01:00`).
The existing cross-midnight logic in `get_profile_events` and `get_event_list` already handles `event_end_time < event_time` — no additional changes needed there.

**Implementation:**
The default is applied in the SQL function after the end-time validation block, before the `INSERT` into `event_mst`. This means all child rows for recurring events also inherit the computed 2-hour end time.

```sql
IF p_event_end_time IS NULL THEN
    p_event_end_time := p_event_time + INTERVAL '2 hours';
END IF;
```
