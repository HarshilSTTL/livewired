### [2026-04-10 02:00] | SP | get_profile_custom_links — fetch custom platform links for a profile

**What changed:**
New read SP that returns all active custom platform links for a given creator profile.
Keeps `get_all_platforms` unchanged — this is a separate dedicated endpoint.

**Params:**
- `p_profile_id uuid` — required; the profile whose custom links to fetch

**Logic:**
- Null guard on `p_profile_id`
- Profile existence check
- SELECT from `profile_custom_links` WHERE `profile_id = p_profile_id AND is_deleted = false`
- ORDER BY `created_at ASC`
- Returns `[]` if no custom links exist

**Response fields:** `custom_id`, `profile_name`, `profile_url`, `is_custom` (always true), `created_at`, `updated_at`

### [2026-04-10 01:00] | TABLE | profile_custom_links — new table for creator custom platform links

**What changed:**
New table to store creator-defined custom platform links (name + URL) per profile.
Custom links are profile-scoped, soft-deleted, and surfaced alongside global platforms
via `get_all_platforms` when called with `p_profile_id`.

**Columns:** `id` (uuid PK) · `profile_id` (FK → creator_profiles) · `profile_name` · `profile_url` · `is_deleted` · `deleted_at` · `created_at` · `updated_at`

### [2026-04-10 00:00] | FIX | show_followers flag ignored in search_profiles, get_profile_by_userid, get_creators

**What changed:**
Three SPs were always returning the follower count regardless of the creator's `show_followers` setting. Fixed by wrapping the `followers` subquery in a `CASE WHEN cp.show_followers = true` check in all three SPs. Additionally fixed `get_creators` which was also counting all follow rows without an `is_active = true` filter.

**Root cause:**
`get_profile_by_id` and `get_profile_by_username` correctly respected the flag, but the fix was never applied to the other three read SPs.

**Fix applied to:**

| SP | Fix |
|---|---|
| `search_profiles` | Added `CASE WHEN show_followers = true` around followers count |
| `get_profile_by_userid` | Added `CASE WHEN show_followers = true` around followers count |
| `get_creators` | Added `CASE WHEN show_followers = true` + added `is_active = true` filter |

**Behaviour after fix:**
- `followers` returns the count when `show_followers = true`
- `followers` returns `null` when `show_followers = false`
