### [2026-04-10 00:00] | SP | update_profile_platforms — manage creator additional platform links

**What changed:**
New write SP dedicated to managing a creator's platform links (channel URLs) independently of the full profile update flow.

**Params:**
- `p_profile_id uuid` — required; the profile to update
- `p_user_id uuid` — required; ownership check
- `p_platforms jsonb` — required; array of `{ platform_id, channel_url, is_default? }` objects

**Logic:**
- Null guards on all three required params
- Ownership check: profile must belong to p_user_id
- Platform validation: all platform_ids must exist in `platforms` table; channel_url must be non-empty for each entry
- Replace-all: DELETE existing `creator_platform_accounts` rows for the profile, then INSERT new ones
- `p_platforms = []` clears all platform links
- Returns success with `profile_id`

**Error cases:** `Profile ID is required` · `User ID is required` · `Platforms list is required` · `Profile not found or access denied` · `One or more platform IDs are invalid` · `Channel URL is required for each platform`
