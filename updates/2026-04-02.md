### [2026-04-02 21:30] | BUG FIX | get_event_list — empty platforms on recurring events

**Root cause:**
- Recurring child rows store no `event_platforms` rows of their own
- Platforms are only stored on the parent event
- SP was using `ep.event_id = e.event_id` (child ID → no rows found → empty `[]`)

**Fix applied (all 4 date branches):**
- `WHERE ep.event_id = e.event_id`
- → `WHERE ep.event_id = COALESCE(e.parent_event_id, e.event_id)`
- For non-recurring events: `parent_event_id = NULL` → `COALESCE` returns `e.event_id` (no change)
- For recurring child rows: `parent_event_id = <parent uuid>` → looks up parent's platforms ✅

**Also fixed (all 4 branches):**
- Key name `'streaming'` → `'platforms'` for consistency across all sections

**Files modified:**
- `functions/events/get_event_list.md`

### [2026-04-02 19:00] | SP | get_event_by_id + update_event + delete_event — new event management SPs

**`get_event_by_id`** (POST /rpc/get_event_by_id)
- Input: `p_event_id uuid`
- Returns full event: title, description, date, time, livestream, video, is_recurring, creator snippet, platforms[], recurring rules
- Child events inherit platforms from parent via `COALESCE(parent_event_id, event_id)`

**`update_event`** (POST /rpc/update_event)
- Inputs: `p_event_id`, `p_user_id` (required) + all other fields optional
- COALESCE pattern — only passed fields are updated
- `p_platforms`: null = don't touch · `[]` = clear all · `[...]` = replace all
- Ownership check: event must belong to a profile owned by p_user_id

**`delete_event`** (POST /rpc/delete_event)
- Inputs: `p_event_id uuid`, `p_user_id uuid`
- Hard delete with ownership check
- CASCADE removes: child occurrences, event_platforms, event_recurring

**Files changed:**
- `functions/events/get_event_by_id.md` — NEW
- `functions/events/update_event.md` — NEW
