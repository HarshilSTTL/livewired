### [2026-04-02 19:00] | SP | get_event_by_id + update_event + delete_event тАФ new event management SPs

**`get_event_by_id`** (POST /rpc/get_event_by_id)
- Input: `p_event_id uuid`
- Returns full event: title, description, date, time, livestream, video, is_recurring, creator snippet, platforms[], recurring rules
- Child events inherit platforms from parent via `COALESCE(parent_event_id, event_id)`

**`update_event`** (POST /rpc/update_event)
- Inputs: `p_event_id`, `p_user_id` (required) + all other fields optional
- COALESCE pattern тАФ only passed fields are updated
- `p_platforms`: null = don't touch ┬╖ `[]` = clear all ┬╖ `[...]` = replace all
- Ownership check: event must belong to a profile owned by p_user_id

**`delete_event`** (POST /rpc/delete_event)
- Inputs: `p_event_id uuid`, `p_user_id uuid`
- Hard delete with ownership check
- CASCADE removes: child occurrences, event_platforms, event_recurring

**Files changed:**
- `functions/events/get_event_by_id.md` тАФ NEW
- `functions/events/update_event.md` тАФ NEW
- `functions/events/delete_event.md` тАФ NEW
