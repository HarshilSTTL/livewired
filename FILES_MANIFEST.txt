API SUMMARY
═══════════════════════════════════════════════════════════════════════════════

Endpoint: POST /rpc/clear_notifications

Parameters:
  • p_user_id (uuid, required)
  • p_notification_ids (uuid[], optional)

Behavior:
  • If p_notification_ids is null: clears ALL notifications for user
  • If p_notification_ids provided: clears only those specific IDs
  • Marks notifications as cleared (doesn't delete from DB)

Response:
  {
    "status": true,
    "message": "X notification(s) cleared"
  }

Security:
  • User isolation enforced (can only clear own notifications)
  • SQL injection prevention (parameterized queries)
  • Idempotent (clearing already-cleared is safe)
